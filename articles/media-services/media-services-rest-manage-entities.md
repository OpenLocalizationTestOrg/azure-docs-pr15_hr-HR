#<a name="managing-media-services-entities-with-rest-api"></a>Upravljanje Media Services entiteti s REST API-JA

> [AZURE.SELECTOR]
- [OSTALE](media-services-rest-manage-entities.md)
- [.NET](media-services-dotnet-manage-entities.md)

Microsoft Azure Media Services je usluzi utemeljenoj na OSTALE utemeljena na OData v3. Zbog toga možete dodati, upit, ažuriranje i brisanje entiteti na približno na isti način kao OData servis. Iznimke će biti oblačićima kada je primjenjivo. Dodatne informacije o OData potražite u članku [protokola Open Data dokumentacije](http://www.odata.org/documentation/).

- Dodavanje entiteti 
- Slanje upita entiteti 
- Prebrojavanje kroz velikih zbirki entiteti
- Ažuriranje entiteti 
- Brisanje entiteti 

>[AZURE.NOTE] Rad s Media Services REST API-JA, primjenjuju se Imajte na umu sljedeće:
>
>Prilikom pristupanja entiteti u Media Services, morate postaviti određene zaglavlje polja i vrijednosti u zahtjevima za HTTP. Dodatne informacije potražite u članku [Postavljanje Media Services REST API razvoj](media-services-rest-how-to-use.md).

>Nakon uspješnog povezivanja s https://media.windows.net, primit ćete 301 preusmjeravanje navodeći drugi URI Media Services. Provjerite naknadni pozivi za nove URI kao što je opisano u [povezuje se s Media Services pomoću REST API -JA](media-services-rest-connect-programmatically.md). 


##<a name="adding-entities"></a>Dodavanje entiteti

Svaki entitet u Media Services dodaje se skup entitet, kao što su resursi, na kroz objavu HTTP zahtjev.

Sljedeći primjer pokazuje kako stvoriti programa AccessPolicy.

    POST https://media.windows.net/API/AccessPolicies HTTP/1.1
    Content-Type: application/json;odata=verbose
    Accept: application/json;odata=verbose
    DataServiceVersion: 3.0
    MaxDataServiceVersion: 3.0
    x-ms-version: 2.11
    Authorization: Bearer http%3a%2f%2fschemas.xmlsoap.org%2fws%2f2005%2f05%2fidentity%2fclaims%2fnameidentifier=youraccountname&urn%3aSubscriptionId=2f84471d-b1ae-4e75-aa09-010f0fc0cf5b&http%3a%2f%2fschemas.microsoft.com%2faccesscontrolservice%2f2010%2f07%2fclaims%2fidentityprovider=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&Audience=urn%3aWindowsAzureMediaServices&ExpiresOn=1337067658&Issuer=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&HMACSHA256=dithjGvlXR9HlyAf5DE99N5OCYkPAxsHIcsTSjm9%2fVE%3d
    Host: media.windows.net
    Content-Length: 74
    Expect: 100-continue
    
    {"Name": "DownloadPolicy", "DurationInMinutes" : "300", "Permissions" : 1}

##<a name="enumerating-through-large-collections-of-entities"></a>Prebrojavanje kroz velikih zbirki entiteti

Prilikom postavljanja upita entiteti, postoji ograničenje od 1000 entiteti vraća odjednom jer javno OSTALE v2 Ograničava rezultate upita 1000 rezultate. Pomoću **preskočiti** i **vrha** prebrojavanje veliku količinu entiteti. 

Sljedeći primjer pokazuje kako koristiti **preskočiti** i **gore** da biste preskočili najprije 2000 zadacima i dobiti sljedeće 1000 zadatke.  

    GET https://media.windows.net/api/Jobs()?$skip=2000&$top=1000 HTTP/1.1
    Content-Type: application/json;odata=verbose
    Accept: application/json;odata=verbose
    DataServiceVersion: 3.0
    MaxDataServiceVersion: 3.0
    x-ms-version: 2.11
    Authorization: Bearer http%3a%2f%2fschemas.xmlsoap.org%2fws%2f2005%2f05%2fidentity%2fclaims%2fnameidentifier=youraccountname&urn%3aSubscriptionId=2f84471d-b1ae-4e75-aa09-010f0fc0cf5b&http%3a%2f%2fschemas.microsoft.com%2faccesscontrolservice%2f2010%2f07%2fclaims%2fidentityprovider=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&Audience=urn%3aWindowsAzureMediaServices&ExpiresOn=1337078831&Issuer=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&HMACSHA256=suFkxhvPWxQVMjOYelOJfYEWkyTWJCBc02pF0N7NghI%3d

##<a name="updating-entities"></a>Ažuriranje entiteti

Ovisno o vrsti entitet i stanje u kojem se nalazi u, možete ažurirati svojstva na taj entitet kroz ZAKRPA, STAVI ili SPAJANJE HTTP zahtjeve za razgovore. Dodatne informacije o te operacije potražite u članku [STAVI/ZAKRPU/PISMA](https://msdn.microsoft.com/library/dd541276.aspx).

Sljedeći primjer koda prikazuje kako ažurirati naziv svojstva entitet resursa.

##<a name="deleting-entities"></a>Brisanje entiteti

Entiteti moguće je izbrisati u Media Services pomoću brisanje HTTP zahtjev. Ovisno o entitet, redoslijed brisanje entiteti možda važna. Na primjer, entiteti kao što su resursi zahtijevaju koje ste opoziv (ili brisanje) sve Locators koje upućuju na tom određenog resursa prije brisanja sredstava.

Sljedeći primjer prikazuje način da biste izbrisali Locator koji je korišten za prijenos datoteke u spremište blobova platforme.

    DELETE https://media.windows.net/API/Locators('nb:lid:UUID:76dcc8e8-4230-463d-97b0-ce25c41b5c8d') HTTP/1.1
    Content-Type: application/json;odata=verbose
    Accept: application/json;odata=verbose
    DataServiceVersion: 3.0
    MaxDataServiceVersion: 3.0
    x-ms-version: 2.11
